# 측정병리(Measurement Pathology) — 지표 설계의 함정과 방어

측정된 지표는 인센티브 구조를 통해 행동을 변경한다. 의도하지 않은 방향으로 최적화되는 5가지 병리를 파악하고 방어해야 한다.

---

## 1. Goodhart의 법칙

### 원래 정의 (Charles Goodhart, 1975)

영국 중앙은행 이코노미스트가 금융정책 관점에서 발견:

> "어떤 변수가 정책의 목표가 되자마자, 그것은 더 이상 좋은 정책 지표(measure)가 되지 않는다."

### Strathern의 일반화 (Marilyn Strathern, 1997)

> "어떤 정량적 지표가 의사결정의 목표가 되자마자, 그것은 사회적 압력을 받게 되고, 측정 도구로서의 정보 가치를 잃는다."

### 약한 형태 (Weak Form) vs 강한 형태 (Strong Form)

| 형태 | 신호 | 예시 | 결과 |
|------|------|------|------|
| **약한 형태** | 지표 최적화가 처음엔 효과 있다가 수렴 | 판매 인센티브: 초반 상승 → 그 후 정체 | 지표 자체는 여전히 신뢰 가능하지만 행동변화 효과 감소 |
| **강한 형태** | 지표 최적화가 본래 목표를 훼손 | 고객 신규 유입 목표: 저질 고객 유입 증가 → 이탈율 급증 | 지표 개선이 실제 성과 악화를 의미 |

### 약한 형태가 작동하는 경우

- 한계수익이 체감하는 상황: "지표 올리기"가 점점 어려워짐
- 동료와 경쟁이 낮을 때: 게임화 유인이 약함
- 단기간 추적 후 폐기될 때: 조작 비용 > 이득

### 강한 형태가 작동하는 경우

- **승자 독식(Winner-Take-All)**: 상위 1위 또는 상위 10%만 보상 → 모두가 극단 최적화
- **가시적 성과 편향**: 측정한 것만 상사에게 보고되고, 측정 안 한 것은 무시
- **인센티브 강도**: 급여·보너스·신용이 지표에 직결될 때

### 방어 전략

| 방어 | 작동 원리 |
|------|---------|
| **다중 지표(Multi-metric)** | 한 지표를 극단 최적화하면 다른 지표가 비명을 지름. 예: 신규고객 ↑ 목표도 있지만, 기존고객 이탈률 ↓ 목표도 동시 추적 |
| **감시와 감사(Audit)** | 지표 정의가 시간에 따라 변경되지 않는가? 계산식을 조작했는가? 정기 감시 |
| **인과 모델링** | 지표를 개선하는 방법과 의도하지 않은 부작용을 사전에 지도. 만약 A를 올리면 B는 어떻게 되는가? |
| **역할 분리** | 지표 설계자와 지표 추적자를 다른 부서로: 마케팅이 자신의 KPI를 리뷰하게 하지 말 것 |
| **정기 재검증** | 1년마다 "이 지표가 여전히 비즈니스 목표를 대표하는가?"를 묻기 |

## 2. Campbell의 법칙

### 정의

> "어떤 정량적 사회지표가 사회적 의사결정을 위해 사용될수록, 더욱 그것은 부패 압력을 받게 되고, 그 지표가 본래 측정하려던 사회 가치 평가를 왜곡시키게 된다."
> — Donald Campbell, 1976

### 핵심 메커니즘

**의사결정 → 측정 → 결과 기준으로 평가 → 조작 유인 발생**

### 현실 사례: NCLB (No Child Left Behind) 교육법

**정책**: 미국 초중고 학교의 성과를 표준화 시험 점수로 측정. 점수 개선 학교는 지원금 증가, 악화 학교는 감시·인수.

**Campbell 효과**:
- 점수 향상에 도움 안 되는 과목(미술, 음악, 체육) 폐지
- 시험에 나올 법한 문제로만 수업 재설계 ("teaching to the test")
- 성적 나쁜 학생을 시험 때 결석 유도하거나 전학 권유
- 점수 조작: 채점 실수 재검토 명목으로 답안 수정
- 학교 현황: 시험 점수는 올랐는데, 창의성·사고력·시민의식 악화

**결론**: 의사결정 후행(lagging)으로서의 "점수" → 의도하지 않은 왜곡.

### 신호 (이것이 Campbell 효과임을 아는 법)

| 신호 | 의미 |
|------|------|
| 지표는 개선되는데 외부 평판 악화 | 지표와 실제 성과의 괴리 |
| 언론/고객이 "이게 말이 되냐"고 비판 | 조작이 보이기 시작함 |
| 지표 계산식이 자주 변경 | 목표를 달성하기 위해 정의를 조정 |
| 좋은 결과를 위한 "창의적 회계" | Campbell 효과의 신호 |

### 방어 전략

| 방어 | 구현 |
|------|------|
| **측정과 결과 분리** | KPI는 추적하되, 성과평가/보상에 직결시키지 않기. "무엇이 일어나고 있는지"와 "누가 책임인지"를 구분 |
| **정기 외부감시** | 내부가 만든 지표만이 아니라, 제3자(고객, 외부 감사, 시장) 평가도 함께 추적 |
| **층화된 지표** | 지표 개선이 다른 계층(예: 단기 점수 vs 장기 능력)에서는 나타나지 않는지 추적 |
| **인센티브 구조 약화** | 지표 달성을 조직원의 월급·보너스와 직결시키지 않기 (정보 역할로만 사용) |

## 3. McNamara의 오류(Fallacy)

### 정의와 4단계 진행

McNamara 오류는 측정 가능한 것만 중요하다고 점차 믿게 되는 4단계 부패를 설명한다:

| 단계 | 행동 | 신호 | 예시 |
|------|------|------|------|
| ① | 측정 가능한 것만 측정 | "이건 수치화할 수 있으니까 하자" | 전쟁: 적군 사망수, 경제: GDP, 교육: 시험점수 |
| ② | 측정 불가능한 것 무시 | "측정할 수 없으니 중요하지 않다" | 사기, 신뢰도, 창의성은 대시보드에 안 나타남 |
| ③ | 측정 불가능한 것을 부중요하다 추정 | "측정할 수 없다 = 무시할 수 있다" | 병사 사기, 문화적 손상, 신뢰 붕괴를 고려 안 함 |
| ④ | 측정 불가능한 것은 존재하지 않는다 선언 | "지표에 안 나타나면 없는 거다" | 정책 입안 시 사기·신뢰도를 빼먹음 |

### 베트남 전 사례 (Robert McNamara, 국방장관)

**목표**: 공산군 약화. **지표**: 매월 적군 사망 수.

**McNamara 오류의 진행**:
1. 월간 사망수를 추적 시작 (①측정 가능)
2. 사망수 증가 = 진전이라고 선포 (②측정 불가능 무시: 베트남 주민 적개심, 미군 병사 사기, 장기 게릴라 능력)
3. 사망수 목표를 설정하고 명령 ("매달 N명 이상 사상")
4. 매달 목표를 채우기 위해 민간인도 센산 가능성 (③ 부중요하다 추정)
5. 지표는 계속 좋아 보였는데 전쟁은 졌음 (④ 단언)

**결과**: 20년 전쟁 후 공산군이 결국 남베트남을 점령. 지표 개선과 실제 목표 달성이 완전히 역행.

### 신호

| 신호 | 의미 |
|------|------|
| "정성적 피드백은 이상한데, 지표는 좋아" | McNamara 진행 중 |
| 지표에 없는 것들이 의사결정에서 간과됨 | ③단계: 부중요성 추정 |
| "이건 측정할 수 없으니까 신경 쓸 필요 없다" 발언 | ④단계: 존재 부정 |

### 방어 전략

| 방어 | 구현 |
|------|------|
| **정성-정량 균형** | 지표만으로 판단하지 않기. 정기적으로 질적 피드백(고객 인터뷰, 팀 설문) 수집 |
| **측정 불가능한 것의 명시적 추적** | "우리는 신뢰도(측정 불가)를 X, Y, Z로 근거 추론하기로 했다"라고 선언하고 추적 |
| **지표 외부 평가자** | 지표 달성을 평가하는 사람과, 현장의 실제 상황을 평가하는 사람을 분리 |
| **정기 "현장 방문"** | 경영진이 직접 현장을 보고, 지표와 현장의 괴리를 체크 |

## 4. Surrogation (대체 지표화)

### 정의

**지표(proxy)와 본래 구성개념(construct)을 혼동하고, 지표를 올리는 것이 목표 달성과 동일하다고 믿는 것.**

#### 비유: "지도는 영토가 아니다"

지표는 지도. 지도를 만드는 것이 영토를 정복하는 것과는 다르다. 하지만 조직이 surrogation에 빠지면, "지도의 특정 부분을 빨갛게 칠하기"가 "실제로 그 영토를 통제하기"와 같다고 믿게 됨.

### Wells Fargo 스캔들 (2016)

**지표**: 고객당 상품 개수 (Cross-sell ratio).

**의도**: "고객에게 더 많은 가치를 제공하자."

**Surrogation의 진행**:
- 처음: Cross-sell ratio 추적 (합리적)
- 중간: 지표 목표를 높임 (7상품/고객)
- 후반: 지표가 목표 자체가 됨 (상품을 많이 파는 것 = 성공)
- 극단: 실제 고객이 필요하지 않은 상품을 무단으로 개설 (계좌 조작, 신용카드 무단 발급)

**결과**: 지표는 폭발적 증가 → 신용도 붕괴, $3B 벌금, CEO 실각. 지표가 구성개념(고객 만족도, 신뢰)과 완전히 역행.

### Surrogation의 5단계 진행

| 단계 | 조직의 상태 | 예시 |
|------|-----------|------|
| ① | 지표 도입 (합리적) | "우리는 고객 만족도를 측정하기 위해 NPS를 추적하자" |
| ② | 지표 최적화 (합리적) | "NPS를 올리기 위해 고객 경험을 개선하자" |
| ③ | 목표로 경화(hardening) | "NPS 목표: 70점. 미달 시 보너스 삭감" |
| ④ | 지표 vs 구성개념 괴리 | NPS는 70 → 실제 이탈율은 증가, 신뢰도는 하락 |
| ⑤ | 게임화 극단화 | 지표를 올리기 위해 본래 목표를 훼손 (고객 호통치기, 수작업 조작) |

### 신호

| 신호 | 의미 |
|------|------|
| "지표를 X% 올려야 한다" vs "고객 만족도를 올려야 한다" 발언의 혼동 | ③단계 |
| 지표는 올랐는데 고객 이탈/만족도는 악화 | ④단계 시작 |
| 지표 달성을 위해 부정행위나 비윤리적 방법 사용 | ⑤단계 |

### 방어 전략

| 방어 | 구현 |
|------|------|
| **프록시 검증 주기** | 1년마다 "이 지표가 여전히 우리 의도를 반영하는가?"를 재평가. (→ references/proxy-metrics.md) |
| **다중 신호(Triangulation)** | 하나 지표가 아니라 여러 지표로 구성개념을 측정. NPS만이 아니라 이탈률, 재구매율, 정성 피드백도 함께 |
| **구성개념 재정의** | 지표 설계 시점에 "우리는 X를 측정하고 싶고, 지표로는 Y를 사용한다. Y ≠ X이므로 한계: Z"라고 명시 |
| **게임화 방지** | 지표를 보상(월급, 보너스)과 직결시키지 않기. "정보" 역할로만 사용 |

## 5. Cobra Effect (역설적 반발)

### 정의

목표 달성을 위한 인센티브가 역설적으로 목표를 악화시키는 현상.

**원래 사례**: 영국령 인도 델리의 독사. 독사를 죽이면 상금을 주는 정책 → 주민이 독사를 대량 양식해서 죽여서 팔기 시작 → 독사 개체 수 증가.

### 적용 메커니즘

```
유인 구조 설계 (A를 올린다) 
  ↓
합리적 자기이익 추구 (A를 올리는 방법을 찾음)
  ↓
예상 밖의 방법 발견 (불의한 방법)
  ↓
목표 달성 (A는 올랐음)
  ↓
본래 목표는 악화 (B는 더 악화됨)
```

### 현실 사례

| 사례 | 유인 | 합리적 대응 | 역설 | 결과 |
|------|-----|-----------|------|------|
| **Enron** | 분기 수익 목표 | 수익 올리기 | 부정 회계 + 자산 거래 반복 | 주가 폭락, 기업 붕괴 |
| **Wells Fargo (재발)** | Cross-sell ratio | 상품 판매 | 무단 계좌 개설 | 신뢰도 붕괴, 벌금 |
| **소프트웨어 버그** | "버그 수 Y개 이하" | 보고 버그 수 줄이기 | 버그 보고를 안 하거나 조작 | 실제 버그는 증가, 품질 저하 |
| **판매 인센티브** | 신규고객 수 X명 | 신규고객 유입 증가 | 저품질 고객 유입 또는 기존 고객 이탈 유도 | 고객 생명주기 가치 악화 |
| **학생 유입** | "신입생 수 N명" | 신입생 모집 | 입학 기준 낮춤 또는 부정 합격 | 졸업률 악화, 평판 손상 |

### 공통점

1. **측정 가능한 것만 인센티브화** — 어렵거나 중요한 것(신뢰, 품질 장기 유지)은 제외
2. **자기기만의 여지** — "기술적으로는 규칙 위반 아니다"라고 정당화 가능
3. **합리적 자기이익** — 개인입장에서는 지극히 합리적
4. **지연된 피해** — 적발까지 수개월~수년 소요

### 신호

| 신호 | 의미 |
|------|------|
| 지표는 개선되는데, 고객 피드백/이탈율은 악화 | 역설적 반발 가능성 |
| "기술적으로는 규칙 위반 아니지만" 같은 발언 | 자기기만이 시작됨 |
| 팀이 창의적으로 지표 달성 방법을 찾을 때 | 아직 양성이지만, 경고 신호 |
| 내부 고발이나 언론 보도 나오기 시작 | 이미 극단화 단계 |

### 방어 전략

| 방어 | 구현 |
|------|------|
| **다중 메트릭** | 한 가지 지표의 극단 최적화를 감시할 "카운터 지표(L4)" 설정 |
| **유인 구조 재설계** | 지표 달성이 아니라, **행동 결과를 인센티브화**. 예: "판매 수"가 아니라 "고객 만족도×리텐션" |
| **합리적 자기이익 가정** | 모든 개인이 수익 극대화를 추구한다고 가정 → "그들이 어떻게 게임화할까"를 사전에 설계 |
| **감시와 감사** | 지표와 실제 행동의 괴리를 정기 감사. "이 증가가 진짜 거기서 왔나?" |
| **내부고발 채널** | 이상한 관행을 신고할 수 있는 심리적 안전장치 |
| **주기적 재검증** | 1년마다 "이 유인 구조가 여전히 행동을 올바른 방향으로 유도하는가" 재평가 |

## 6. Jerry Muller의 지표 집착(Metric Fixation)

### 10가지 의도하지 않은 결과

Muller는 저서 《The Tyranny of Metrics》에서 지표 기반 관리의 광범위한 부작용을 문서화:

| # | 현상 | 메커니즘 | 사례 |
|---|------|---------|------|
| 1 | 목표 치환 | 수단(지표)이 목표(본래 목적)가 됨 | 판매 수 > 고객 만족 |
| 2 | 단기주의 | 장기 이익을 희생하고 분기 지표 최적화 | R&D 투자 削 → 단기 이익 ↑ |
| 3 | 직원 시간 비용 | 지표 추적·리포팅에 소요되는 시간 급증 | 의료: 차트 작성 > 환자 진료 |
| 4 | 규칙 연쇄(Rule cascade) | 지표 게임화 방지를 위해 규칙 추가 → 복잡도 폭증 | 은행: 컴플라이언스 규칙 수백 개 |
| 5 | 운 보상(Rewarding luck) | 외부 요인(시장 호황, 운)이 지표에 과다 가중 | 판매량 많은 지역 리더 = 능력 있는가? |
| 6 | 위험 기피 | 지표가 나쁘면 채용/투자를 피함 → 혁신 정체 | 실험적 프로젝트 외면 |
| 7 | 협력 감소 | 팀 간 경쟁으로 정보 공유 감소 | A팀 비법 = B팀의 손실 |
| 8 | 작업 품질 저하 | "숫자만 맞추기"로 질 무시 | 의료: 진료 건수 ↑ + 환자 결과 ↓ |
| 9 | 생산성 역설(Productivity paradox) | 지표 추적으로 행정 비용 증가 > 효율 개선 이득 | IT 투자 > 생산성 개선 미미 |
| 10 | 부등(Inequity) | 측정 어려운 직군/지역 무시 | 정량 부서 우대, 정성 부서 차별 |

### 합성 신호 (Meta-symptom)

조직에서 다음이 동시에 나타나면 지표 집착이 심한 것:

```
지표 수 증가 + 행정 비용 증가 + 직원 피로도 증가 + 혁신 감소
```

### 방어: Muller의 처방

**지표는 **판단(judgment)의 입력**이지, 판단 자체가 아니다.**

| 원칙 | 구현 |
|------|------|
| **지표 수 제한** | 총 15개 이내. 많을수록 정보가 아니라 노이즈 |
| **정성 판단 복권** | 리더가 지표 밖에서 판단할 권리를 명시적으로 허락 |
| **지표 회의주기 제한** | 주간 리뷰는 필요하지만, 일일 스탠드업에서 지표 남용 금지 |
| **직원 자율성** | 지표에 가중 신경 쓰도록 강제하지 않기. 일의 내재 동기 보호 |
| **정기 재평가** | 분기마다 "이 지표가 정말 필요한가?" 물어보기 |

| **McNamara** | 중요한 것을 측정 안 하나? | 약/중/강 | 정성 피드백 채널? 예/아니오 |
